# 执行计划：仓库整仓审查、修复与文档治理

## 目标

对当前仓库进行完整审查，修复确认的问题，优化目录结构和测试组织，清理低价值或失真文档，并补齐目录结构说明文档，使后续开发者可以快速接手。

## 范围

- 代码审查与必要修复
- 测试目录与仓库结构优化
- 文档索引、产品口径、架构口径与说明文档同步
- 新增目录结构说明

## 不在范围

- 新增任务、实验、评测、成绩等新业务功能
- 把当前模块化单体改成 Maven 多模块或微服务
- 引入新的基础设施集成

## 风险

- 代码本身已经通过验证，本轮主要风险在于文档和目录调整引入路径漂移。
- 仓库中存在大量 `.agents/skills/**` 外部改动，本轮需要避免误改非项目核心文件。

## 验证路径

- `./mvnw spotless:apply`
- `./mvnw verify`
- 人工复核 `README.md`、`ARCHITECTURE.md`、`docs/index.md` 和新增结构说明文档的链接关系

## 决策记录

| 决策 | 原因 |
|------|------|
| 将集成测试从 `api` 包迁到 `integration` 包 | 测试分类不应伪装成生产分层 |
| 删除弱价值文档入口 | 减少接手成本，避免读到与当前开发无关的内容 |
| 新增仓库结构说明文档 | 让后续开发者能快速定位代码、测试、文档与工作记忆文件 |
