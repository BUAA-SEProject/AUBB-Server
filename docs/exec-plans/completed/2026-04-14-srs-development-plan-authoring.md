# 基于 SRS 编写开发总计划

## 目标

基于 AUBB 软件需求规格说明书，为当前后端仓库产出一份可执行的 `docs/plan.md`，把系统级需求转化为后端实施阶段、交付门槛和开发顺序。

## 已完成工作

- 读取 SRS、当前架构文档和计划入口文档
- 提取 V1 / V1.1 / 后续版本范围
- 提取功能分组、非功能需求、验收基线和未决事项
- 编写 `docs/plan.md`
- 更新 `docs/plans.md` 以链接总计划
- 将 `docs/plan.md` 接入 harness 文档校验

## 关键决策

- 将 `docs/plan.md` 作为仓库级总计划入口，而不是继续堆叠到 `docs/plans.md`
- 计划按阶段、横切保障、里程碑和风险组织，而不是仅按功能罗列
- 计划以当前后端仓库为主，前端和外部系统只保留协同边界

## 验证路径

- 执行 `.\mvnw.cmd verify`
- 确认文档链接解析通过
- 确认 `docs/plan.md` 进入文档校验范围
