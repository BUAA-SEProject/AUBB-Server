# 执行计划：Assignment 第一切片

## 目标

在当前课程系统基础上新增 `assignment` 模块，打通教师创建、发布、关闭任务，以及学生查看任务的最小闭环。

## 范围

- `assignment` 数据模型、迁移、Mapper、应用服务、控制器
- 教师侧任务管理接口
- 学生侧任务查看接口
- 审计动作、数据库结构说明、产品规格与架构文档同步

## 不在范围

- 提交、评测、成绩、Rubric、附件、资源正文
- 任务模板复用
- 消息队列或异步编排

## 风险

- 任务可见性需要与教学班成员关系保持一致，不能只按课程维度放宽。
- 第一切片若过度设计，会提前耦合 `submission / judge / grading`。

## 验证路径

- 任务创建/发布/关闭集成测试
- 学生任务查看与越权集成测试
- `./mvnw spotless:apply`
- `./mvnw clean verify`

## 决策记录

| 决策 | 原因 |
|------|------|
| 新建 `modules.assignment` | 为后续提交、评测、批改留下稳定边界 |
| 任务允许可选绑定教学班 | 同时覆盖课程级任务和班级级任务 |
| 状态只保留 `DRAFT / PUBLISHED / CLOSED` | 先做最小闭环 |
