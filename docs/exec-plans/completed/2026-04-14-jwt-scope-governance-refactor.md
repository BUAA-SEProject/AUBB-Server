# 执行计划：JWT、分层管理员与即时生效配置重构

## 目标

将当前 Phase 2 平台治理实现重构为新的治理模型：

- 使用 JWT Bearer Token 认证
- 固化学校/学院/课程/班级四层组织
- 使用支持多身份的作用域管理员模型
- 移除平台配置版本化，改为单份配置即时生效

## 范围

- 仓库内代码、测试、迁移、架构与产品文档
- 上级 `../docs` 中与平台治理、认证和 API 直接相关的文档

## 不在范围

- 刷新令牌、单点登录正式接入、令牌黑名单
- 课程域权限模型的完整实现
- 平台自举 UI 和首个学校节点自动初始化

## 设计决策

1. 认证改用 Spring Security Resource Server + 本地签发 JWT。
2. 平台治理身份使用 `user_scope_roles`，每条记录绑定一个组织节点。
3. 组织树固定为 `SCHOOL -> COLLEGE -> COURSE -> CLASS`。
4. 平台配置只保留单份生效配置，修改即覆盖并立即生效。
5. 审计查询先收敛为学校管理员能力。

## 风险

- 旧接口和测试对 Session / CSRF 有强依赖，重构范围较大。
- 旧文档对角色和配置版本化描述较多，存在较高的同步成本。
- Spring Security 7 的 JWT 配置 API 与旧版本差异较大，需严格按官方方式装配。

## 验证路径

- 领域测试：组织层级策略、身份作用域策略、密码策略、JWT 令牌属性
- 接口测试：登录、Bearer 访问、未授权/越权、配置即时更新、组织创建、用户导入与身份分配
- 全量验证：`mvnd verify`

## 退出条件

- 旧 Session 能力和配置版本化能力已从代码与文档中移除
- 新的 JWT、分层组织、多身份治理能力均有自动化测试证据
- 文档、迁移和实现语义一致
