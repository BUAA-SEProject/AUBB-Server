# 执行计划：课程系统第一切片

## 目标

基于 `design/courses.md` 在 AUBB-Server 中交付课程系统第一切片，使平台从“平台治理与 IAM”扩展到“课程模板、开课实例、教学班、课程成员和班级功能开关”的实际可用状态。

## 范围

- `src/main/resources/db/migration` 下课程域表结构
- `src/main/java/com/aubb/server/modules/course` 及必要的组织/认证协作代码
- `src/test/java/com/aubb/server/api` 下课程系统集成测试
- 仓库内外与课程设计、数据库和 API 直接相关的文档

## 不在范围

- 课程资源正文、公告正文、讨论贴、任务、提交、判题、成绩
- 学生自主选课、邀请码加课、选组与退组选课流程
- 通用 ABAC 引擎、课程级细粒度 staff scope 自定义配置
- 异步 outbox、事件总线和多模块拆分

## 本轮落地重点

1. 课程主数据
   - 学期
   - 课程模板
   - 课程开设实例
2. 组织与治理联动
   - 课程开设实例自动绑定 COURSE 组织节点
   - 教学班自动绑定 CLASS 组织节点
   - 跨学院共同管理映射
3. 课程成员
   - 教师、助教、学生成员关系
   - 教师批量添加与 CSV 导入既有系统用户
   - 成员关系同步到 `user_org_memberships`
4. 教学班能力
   - 多年份班级
   - 班级功能开关
5. 课程权限
   - 教师可管理课程与班级
   - 助教权限显式受限
   - 同一用户可在不同班级承担不同课程角色

## 风险

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 一次性照搬 `courses.md` 设计会导致实现面过大 | 交付失控 | 先压缩为课程第一切片，只实现与当前主链路直接相关的表和接口 |
| 课程成员与平台治理身份混淆 | 授权边界不清 | 明确课程权限完全基于 `course_members`，不复用平台治理身份 |
| 课程/班级组织节点由课程模块创建时容易与组织治理规则冲突 | 数据不一致 | 通过组织应用服务创建节点，并把课程模块视为受控调用方 |

## 验证路径

- `CourseSystemIntegrationTests`
  - 学校/学院治理与课程创建协同成立
  - 课程可跨学院共同管理
  - 教师可创建不同年份班级并切换班级功能
  - 教师可批量添加/导入成员
  - 助教可在一个班级当学生、另一个班级当助教，且权限受限
- `./mvnw verify`

## 决策记录

- 课程系统当前以 `modules.course` 聚合模块实现；若后续复杂度继续增长，再拆成更细课程子模块。
- 学生自主选课入口本轮不实现，课程成员只能由教师从系统既有用户中批量添加或导入。

## 退出条件

- 课程第一切片的数据库结构已经通过 Flyway 落地
- 课程模板、学期、开课实例、教学班、成员和班级功能开关接口已经可用
- 助教多角色与权限边界已经有自动化测试证据
- 架构、规格、数据库和 API 文档已同步
- `./mvnw verify` 通过
