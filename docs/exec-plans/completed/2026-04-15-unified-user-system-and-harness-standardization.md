# 执行计划：统一用户系统建设与 Harness 规范化

## 目标

围绕 AUBB 平台治理切片，完成用户系统设计收敛、TDD 驱动的实现补强、harness 文档治理以及仓库内外文档统一，形成适合继续承接课程域能力的稳定工程基线。

## 范围

- `AGENTS.md` 与 `docs/` 下的 harness / 开发规范文档
- 当前 Phase 2 用户、认证、组织、授权相关代码与测试
- `docs/` 与 `../docs/` 中与用户系统、认证授权、工程规范、数据库、API 和用户手册直接相关的文档

## 不在范围

- 课程成员模型、课程资源、任务、提交、判题与成绩模块的完整实现
- 刷新令牌、统一认证接入、服务端令牌黑名单
- 平台概览、异常事件中心与前端页面实现

## 计划动作

1. 先用测试定义缺口，补齐用户详情、组织摘要、账号状态与会话超时规则。
2. 在不突破 Phase 2 边界的前提下，完善用户查询与管理返回模型。
3. 规范 `AGENTS.md` 与 `docs/` 下的长期文档，明确 TDD、Skills、fmt、注释与文档治理规则。
4. 清理不应留在全局文档区的过程性内容。
5. 同步更新 `../docs/` 的需求、设计、API、工程规范、数据库与用户手册口径。

## 风险

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 误把课程域角色提前混入平台治理身份 | 后续课程模块耦合加重 | 文档中显式拆分“平台治理身份”和“课程成员角色” |
| 修改用户返回模型时破坏现有接口测试 | 引发回归 | 先写失败测试，再最小实现并回归验证 |
| 删除或移动过程性文档时造成链接失效 | harness 校验失败 | 同步更新索引与链接校验，再执行全量验证 |

## 验证路径

- 用户系统测试：
  - 管理员可查询用户详情并看到归属组织摘要、身份、状态字段
  - `DISABLED` / `EXPIRED` 用户无法登录
  - JWT 默认会话时长具有自动化测试证据
- 文档一致性：
  - `docs/` 只保留长期有效文档
  - `../docs/` 与当前实现的认证、组织、用户系统口径一致
- 全量验证：
  - `./mvnw test`
  - `./mvnw verify`

## 退出条件

- 平台治理阶段的用户系统边界、查询能力与文档口径已经收敛
- `docs/` 下不再保留过程性工作记忆，仓库级 harness 已有自动化约束
- TDD、Skills、fmt、中文注释和文档同步规则已进入仓库长期规范
- `./mvnw verify` 通过
