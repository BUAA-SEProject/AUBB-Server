# 执行计划：Submission 第一切片

## 目标

在当前 assignment 第一切片基础上新增 `submission` 模块，打通学生正式提交、学生查看本人提交，以及教师按作业查看提交的最小闭环。

## 范围

- `submission` 数据模型、迁移、Mapper、应用服务、控制器
- 学生侧正式提交与本人提交查询接口
- 教师侧按作业查看提交接口
- 审计动作、数据库结构说明、产品规格与架构文档同步

## 不在范围

- 在线工作区、草稿恢复、试运行会话
- 文件上传、工程快照、附件存储
- 自动评测、成绩、Rubric、人工批改

## 风险

- 提交受理必须严格复用 assignment 可见性和时间窗口规则，不能出现“可看不可提”或“已关闭仍可提”的错位。
- 第一切片若过度设计，会提前耦合工作区、运行和评测模型。

## 验证路径

- 学生正式提交集成测试
- 提交次数限制和时间窗口边界测试
- 教师查看提交与学生越权测试
- `./mvnw spotless:apply`
- `./mvnw clean verify`

## 决策记录

| 决策 | 原因 |
|------|------|
| 新建 `modules.submission` | 为后续试运行、评测和成绩留下稳定边界 |
| 第一切片只做正式提交 | 先建立可靠提交受理模型 |
| 提交内容先采用文本正文 | 不提前引入文件与快照复杂度 |
| 提交状态只保留 `SUBMITTED` | 先做最小闭环，评测状态后续再扩展 |
