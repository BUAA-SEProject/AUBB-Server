# 2026-04-16 成绩册导出与统计报告第一阶段

## 目标

在不新增成绩事实表的前提下，把教师侧成绩能力从“可浏览”推进到“可导出 / 可统计”：为 offering / class 成绩册补齐 CSV 导出，以及基于现有成绩矩阵的统计报告第一阶段。

## 范围

- 教师 / 管理员按开课实例导出 CSV 成绩册
- 教师 / 管理员 / 班级助教按教学班导出 CSV 成绩册
- 教师 / 管理员按开课实例查看统计报告
- 教师 / 管理员 / 班级助教按教学班查看统计报告
- 统计报告覆盖总体摘要、按作业统计和按班级对比
- 继续复用“每个学生每个作业最新一次正式提交”的聚合语义

## 不在范围

- 多作业加权总评
- 学生侧成绩导出
- 更复杂的多维报表与长期趋势分析
- 新增成绩专用事实表

## 风险

- CSV 导出和页面成绩册若聚合逻辑分叉，会造成教师下载结果与页面视图不一致。
- 班级责任 TA 只能导出 / 查看自己负责教学班的数据，不能借此越权访问 offering 级成绩。
- 课程公共作业和班级专属作业混合时，导出矩阵和统计口径必须维持一致的“适用范围”语义。

## 验证路径

- `bash ./mvnw -Dtest=GradebookIntegrationTests,GradingIntegrationTests test`
- `bash ./mvnw clean verify`

## 决策记录

- 导出与统计继续放在 `grading` 模块内，复用现有成绩册读模型。
- offering / class 两类 CSV 导出都基于已有成绩册快照生成，不额外查询第二套历史矩阵。
- offering 级统计报告额外返回按班级对比；class 级统计报告只返回本班总体摘要和按作业统计。
