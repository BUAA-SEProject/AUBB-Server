# 安全

## 当前基线

- Spring Security 默认启用。
- `/actuator/health` 和 `/actuator/info` 出于运维检查目的，被显式设为公开。
- OpenAPI 发现端点当前保持公开，以支持早期后端开发。
- 在明确的认证模型落地前，其他所有路由默认都要求认证。

## 规则

1. 不要引入匿名可访问的业务端点，除非已经在文档中说明原因。
2. 密钥和凭证不得进入版本库，优先采用环境变量或外部配置。
3. 新增对外部系统的调用时，要记录所需凭证和集成边界。
4. 任何会影响认证或授权行为的变更，都必须在同一改动中更新本文件。

## 后续事项

在发布任何面向客户的 API 之前，先定义清晰的认证与授权模型。
